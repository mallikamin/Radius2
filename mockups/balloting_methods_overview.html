Plot Balloting Mockups - Method Comparison

This page helps management review how each method would look in the admin system before deciding. Language is non-technical so operations and leadership can align quickly.

Project: Sitara Square

Draft Date: 03 Mar 2026

Audience: COO / Admin / Ops

Option A - Simple Shuffle

Recommended v1

Most straightforward. We shuffle eligible buyers and eligible plots fairly, then pair them one by one.

  • Clear to explain publicly
  • Strong legal defensibility
  • Fast implementation
Open Detailed Mockup

Option B - Lottery Draw

Live Event Friendly

Same fairness as A, but visualized as a live "next draw" event on big screen for audience confidence.

  • Best for stage experience
  • Public trust via transparency
  • Slightly more UI work
Open Detailed Mockup

Option C - Weighted

Policy Sensitive

Buyers get higher chance based on approved policy (for example: full payment, early EOI).

  • Rewards business priorities
  • Needs very clear policy communication
  • Higher dispute risk if not explained well
Open Detailed Mockup

Option D - Tiered

Balanced

Two rounds: premium plots with priority buyers first, then general draw for everyone else.

  • Fairness + business control balance
  • Easy narrative if policy is pre-announced
  • More operations steps than A/B
Open Detailed Mockup

Common Flow Across All Methods

1. Create Event
2. Load Marla Inventory
3. Mark Selected Plots
4. Freeze Eligible Buyers
5. Run Draw
6. Publish & Print

Governance guardrails in every option: snapshot lock, audit trail, reason-required reserved plots, signed result export.
